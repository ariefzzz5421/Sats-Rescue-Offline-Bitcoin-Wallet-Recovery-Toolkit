# Sats Rescue

**Offline Bitcoin Wallet Recovery Toolkit**

Sats Rescue helps Bitcoin users recover access to their own wallet when they still have a seed phrase, extended private key (`xprv/yprv/zprv`), raw private key, or WIF, but the wallet app shows the wrong address type or import format.

It runs locally, uses no network, and has no third-party Python dependencies.

> This tool does not recover unknown/lost seed phrases. It does not brute-force missing words.

## What This Tool Can Do

- Find which standard Bitcoin derivation path creates a known address.
- Convert raw hex private keys to WIF.
- Convert HD private keys like `xprv`, `yprv`, and `zprv` to the matching child key.
- Show Electrum sweep lines such as `p2wpkh:...` for Native SegWit addresses.
- Help users move funds to a fresh wallet they control.

## What This Tool Cannot Do

- Guess a lost seed phrase.
- Recover funds without a valid private key, extended private key, or seed phrase.
- Bypass exchange, wallet, or blockchain rules.
- Protect funds if the seed/private key was generated by a malicious bot.

## Safety Rules

Never paste seed phrases, WIF, private keys, or sweep lines into chat, websites, Discord, Telegram, or GitHub issues.

Safe to share publicly:

- Public BTC address.
- Transaction ID.
- Wallet app name.
- Error message.
- Key format only, for example `starts with zprv`, `12-word seed`, `WIF starts with K`.

Not safe to share:

- Seed phrase.
- Private key.
- WIF.
- Extended private key.
- Electrum sweep line.
- Screenshots that show any secret.

## Install

Clone the repo:

```bash
git clone https://github.com/YOUR_USERNAME/sats-rescue.git
cd sats-rescue
```

Check Python:

```bash
python --version
```

Python 3.10+ is recommended. No package installation is required.

For maximum safety, disconnect from the internet before entering secrets.

## Recover From Seed Phrase

Use this when you have a BIP39 seed phrase and want to find the path/address.

```bash
python btc_seed_recovery_tool.py --address YOUR_PUBLIC_BTC_ADDRESS --limit 20
```

When prompted, paste the seed phrase. The input is hidden.

If your seed uses an extra BIP39 passphrase, add:

```bash
python btc_seed_recovery_tool.py --address YOUR_PUBLIC_BTC_ADDRESS --limit 20 --passphrase
```

If it finds a match, run again offline:

```bash
python btc_seed_recovery_tool.py --address YOUR_PUBLIC_BTC_ADDRESS --limit 20 --show-private
```

Then use the `Electrum sweep line` in Electrum:

```text
Wallet > Private Keys > Sweep
```

## List First Addresses From a Seed

If you do not know the target address, list common first addresses:

```bash
python btc_seed_recovery_tool.py --list --limit 3
```

This checks common Bitcoin standards:

- `m/44'/0'/0'/0/i` Legacy P2PKH, address starts with `1`
- `m/49'/0'/0'/0/i` Nested SegWit, address starts with `3`
- `m/84'/0'/0'/0/i` Native SegWit, address starts with `bc1q`
- `m/86'/0'/0'/0/i` Taproot, address starts with `bc1p`

## Recover From xprv / yprv / zprv

Use this when the secret starts with `xprv`, `yprv`, or `zprv`.

```bash
python btc_hd_recovery_tool.py --address YOUR_PUBLIC_BTC_ADDRESS --limit 20
```

If your extended key is already at address depth, try:

```bash
python btc_hd_recovery_tool.py --address YOUR_PUBLIC_BTC_ADDRESS --path m --show-private
```

If it finds a match, use the printed Electrum sweep line.

## Recover From Single Private Key / WIF

Use this when you have one raw private key or WIF.

```bash
python btc_key_recovery_tool.py --address YOUR_PUBLIC_BTC_ADDRESS
```

If it matches:

```bash
python btc_key_recovery_tool.py --address YOUR_PUBLIC_BTC_ADDRESS --show-private
```

## Address Type Guide

| Address starts with | Type | Electrum sweep hint | CEX network |
| --- | --- | --- | --- |
| `1` | Legacy / P2PKH | plain WIF | BTC / Bitcoin Mainnet |
| `3` | Nested SegWit / P2SH-P2WPKH | `p2wpkh-p2sh:WIF` | BTC / Bitcoin Mainnet |
| `bc1q` | Native SegWit / P2WPKH | `p2wpkh:WIF` | BTC / Bitcoin Mainnet |
| `bc1p` | Taproot / P2TR | wallet must support Taproot | BTC / Bitcoin Mainnet |

Do not choose BEP20, ERC20, BSC, TRC20, or Lightning unless you intentionally use that separate network and understand the risk.

## Recommended Recovery Flow

1. Use the tool offline to confirm the secret matches the public address.
2. Create a fresh wallet with a new seed phrase in Electrum, Sparrow, or a hardware wallet.
3. Sweep/send all funds from the recovered key into the fresh wallet.
4. Wait for confirmation.
5. If sending to an exchange, use only the exchange's `Deposit BTC` address on `Bitcoin Mainnet`.
6. Stop using the recovered/bot-generated wallet.
